# Phase 8 Plan 1: AI Document Drafting with Gemini Streaming

## Objective
Build a "Draft Document" modal accessible from entity detail pages that uses Gemini to generate grounded AI drafts of compliance filings and intercompany agreements, with streaming output and legal disclaimer.

## Context
- @src/app/api/ai/briefing/route.ts — Existing Gemini integration pattern
- @src/components/entities/entity-detail-tabs.tsx — Entity detail tabs to add button
- @src/lib/db/types.ts — Entity, Director, Jurisdiction types

## Tasks

### Task 1: API route and draft modal component
type="auto"

Create `/api/ai/draft` POST route that:
- Accepts entity ID and document type (compliance_filing | intercompany_agreement)
- Fetches entity data (name, jurisdiction, incorporation date, directors, agreements)
- Builds grounded prompt with entity context
- Streams Gemini response using @google/genai streaming API
- Returns ReadableStream for client consumption
- Template fallback when no API key

Create DraftDocumentModal component:
- Trigger button "Draft Document" on entity detail page
- Document type selector (compliance filing, intercompany agreement)
- Streaming output display area
- "AI draft -- requires legal review" disclaimer
- Loading and error states

Done criteria:
- Modal opens from entity detail
- Selecting type and clicking generate triggers streaming AI response
- Entity data appears in generated draft
- Disclaimer visible on every draft
- API key stays server-side

## Verification
- Modal accessible from entity detail page
- Draft grounded in entity data (name, jurisdiction, directors visible in output)
- Disclaimer present
- No API key in client bundle

## Output
- src/app/api/ai/draft/route.ts
- src/components/entities/draft-document-modal.tsx
- Updated entity-detail-tabs.tsx with Draft Document button
