  <action>
    Create `src/lib/compliance-engine/alert-aggregator.ts`:

    Input type for aggregation (entity context needed for alert display):
    ```typescript
    interface EntityScoreWithContext {
      score: EntityRiskScore
      entity: { id: string, name: string, legalName: string }
      jurisdiction: { countryCode: string, countryName: string }
    }
    ```

    `aggregateAlerts(entityScores: EntityScoreWithContext[]): AlertItem[]`

    Logic:
    1. For each entity score where riskLevel !== 'ok':
       - For each overdue deadline: create AlertItem with alertType='overdue', urgencyScore = daysUntilDue (negative = most urgent)
       - For each dueSoon deadline: create AlertItem with alertType='due_soon', urgencyScore = daysUntilDue
       - If isDissolving and no overdue: create AlertItem with alertType='at_risk', urgencyScore = 0
    2. Generate human-readable messages:
       - Overdue: "{requirementType} for FY{year} is {days} days overdue. Immediate action required."
       - Due soon: "{requirementType} for FY{year} due in {days} days."
       - At risk: "{entityName} is dissolving. Review compliance obligations."
    3. Sort all alerts by urgencyScore ascending (most negative first = most overdue)
    4. Return AlertItem[]

    Format requirement_type for messages: 'annual_filing' -> 'Annual filing', 'tax_return' -> 'Tax return', 'agent_renewal' -> 'Agent renewal'.

    Create test file with pinned mock data.
