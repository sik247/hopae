  <action>
1. Install dev dependencies: `npm install --save-dev tsx dotenv`

2. Add to package.json scripts: `"db:seed": "tsx scripts/seed.ts"`

3. Add `SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here` to `.env.example`

4. Create `scripts/seed.ts` with the following structure:

**Top of file:**
- Import `createClient` from `@supabase/supabase-js`
- Import `config` from `dotenv` and call `config({ path: '.env.local' })`
- Create supabase client using `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`
- Define `SEED_DATE = new Date('2026-03-13')` and `daysFromSeed(days: number): string` helper

**Deterministic IDs:**
- Define `IDS.JURISDICTION` object with 23 UUIDs (format: `a0000000-0000-0000-0000-000000000NNN`)
- Define `IDS.ENTITY` object with 63 UUIDs (format: `b0000000-0000-0000-0000-000000000NNN`)
- Export IDS so Plan 02 can reference them if needed

**JURISDICTIONS array (23 rows):**
Use the exact jurisdiction roster from RESEARCH.md. Each jurisdiction must have:
- `id`, `country_code`, `country_name`, `currency`, `timezone`
- `filing_rules` JSONB with: `annual_filing_month`, `tax_deadline_doy`, `agent_renewal_month`, `fiscal_year_end_month`, `fiscal_year_end_day`, `grace_period_days`, `notes`
- CRITICAL: Japan/India/Australia have non-Dec-31 fiscal year ends (JP: Mar 31, IN: Mar 31, AU: Jun 30). Set `fiscal_year_end_month` correctly per RESEARCH.md.

**ENTITIES array (63 rows, inserted in FK-safe order):**
- Insert Luxembourg HQ FIRST (parent_entity_id: null, entity_purpose: 'hq')
- Then 23 provider_key entities (parent_entity_id: LU_HQ ID)
- Then 28 customer_entity entities (parent_entity_id: LU_HQ ID)
- Then 11 branch entities (parent_entity_id: their SUBSIDIARY's ID, NOT HQ)
- Each entity has: name, legal_name (with correct local suffix from RESEARCH.md), entity_type, entity_purpose, jurisdiction_id, parent_entity_id, incorporation_date, registration_number (generate plausible format per country), status (active for most; 'dissolving' for Hopae Belgium Ghent Branch and Hopae Estonia Branch), banking_info JSONB, registered_agent JSONB, metadata JSONB (include eID system name for provider_key entities)
- Use the exact entity roster from RESEARCH.md (all 63 entities with correct names, dates, statuses)

**Banking info per entity:**
Use country-appropriate bank names from RESEARCH.md. Generate structurally plausible (but fake) IBANs. Set currency to match jurisdiction.

**Registered agent per entity:**
Use the agent roster from RESEARCH.md. Set renewal_date to a future date for active entities, past date for dissolving entities.

**main() function:**
- Truncate all 7 tables in reverse FK order using `.delete().neq('id', '00000000-0000-0000-0000-000000000000')`
- Insert jurisdictions, then entities (HQ first via separate insert, then rest)
- Log counts after each insert
- Exit with code 1 on any error

Do NOT seed directors, compliance_requirements, intercompany_agreements, or alerts yet — those come in Task 2 and Plan 02.
